# 全局选项

窗口**底部**有两项全局设置，对**几乎所有功能**生效（在子命令参数之前传给 siglus-ssu）。

---

## 纯 Python 模式（--legacy）

| 项目 | 说明 |
|---|---|
| 位置 | 底部左侧复选框 |
| 默认 | **不勾选** |
| 作用 | 禁用 Rust 原生加速，全部用纯 Python 实现 |

### 什么时候勾选

- 日志出现与 `native_accel`、Rust 扩展相关的错误
- 便携版或环境无法加载原生扩展时

### 缺点

- 大任务（提取、编译大包）会**明显变慢**

### 汉化时建议

- 正常情况下**不要勾选**
- 仅作排查手段

---

## 常量配置（--const-profile）

| 项目 | 说明 |
|---|---|
| 位置 | 底部「常量配置」下拉框 |
| 可选值 | **0**（默认）、**1**、**2** |
| 作用 | 切换内置 const.py 配置档，影响编译器表单/操作码表 |

### 什么时候改

- **编译** 时在日志报 form / element / opcode 相关错误
- 确认游戏引擎变体与默认 profile 不一致（少见）

### 汉化时建议

- 保持 **0**，只有编译失败且确认与常量有关时再试 1 或 2
- 不能与编译的 **增量缓存（--tmp）** 同时使用（见编译教程高级选项）

---

## 与面板选项的关系

执行时实际命令形如：

```
siglus-ssu [--legacy] [--const-profile N] <功能参数…>
```

日志第一行会以 `$ …` 显示完整命令，便于核对。

---

## 常见错误

| 现象 | 处理 |
|---|---|
| 勾选 legacy 后仍失败 | 问题多半不在 Rust，查具体功能教程与故障排除 |
| profile 1/2 仍编译失败 | 检查 .ss 语法、编码；查阅 manual_cn.md |
| 改了 profile 忘记改回 | 完成后改回 0，避免影响其他游戏 |
