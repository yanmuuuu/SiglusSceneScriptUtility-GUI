# 编译 · 使用教程

汉化**最后核心步**：把改好的脚本目录重新打包为 `.pck`，供游戏加载。

---

## 这个功能做什么

- **标准编译**：工作目录中的 `.ss` → 生成新的 `.pck`
- **仅编译 Gameexe.dat**：只处理游戏全局配置

---

## 支持的主要文件/目录

| 类型 | 说明 |
|---|---|
| 输入目录 | 含 `.ss` 源码、`Gameexe.ini`（如有）、提取产生的目录结构 |
| 输出 | `.pck` 文件路径，或已存在文件夹（生成 Scene.pck） |

---

## 逐步操作（汉化：标准编译）

1. 左侧 **场景与脚本** → **编译**
2. **操作类型** → **标准编译（.ss → .pck）**
3. **输入目录** → 选文件夹 → `汉化工作目录\output_xxx`（含改好的 .ss）
4. **输出** → **另存为…** → `汉化工作目录\SceneZH.pck`（名称自定）
5. **源文件编码** → **UTF-8**（与编辑器保存编码一致）
6. **开始执行**
7. 日志出现 `=== Compiling Stats ===` 即成功

---

## 字段说明

| 字段 | 支持什么 | 必填 | 留空/默认 | 结果 |
|---|---|---|---|---|
| 操作类型 | 标准 / 仅 Gameexe | 必选 | — | 决定编译范围 |
| 输入目录 | 含 .ss 的工作目录 | **必填** | — | 编译源 |
| 输出 | .pck 路径或文件夹 | **必填** | — | SceneZH.pck 等 |
| 源文件编码 | 自动 / UTF-8 / Shift-JIS | 建议选 UTF-8 | 自动检测 | 读 .ss 时用 |
| 保留临时文件 --debug | 复选框 | 否 | 不保留 | 调试用临时目录 |
| 串行编译 --serial | 复选框 | 否 | 并行 | 慢机可勾选 |
| 高级 · dat-repack | 仅重打包已有 .dat | 否 | — | 特殊流程 |
| 高级 · 禁用加密 --no-angou | 复选框 | 否 | 正常加密 | 特殊测试 |
| 高级 · 禁用 LZSS --no-lzss | 复选框 | 否 | 正常压缩 | 特殊测试 |
| 高级 · 增量缓存目录 --tmp | 文件夹路径 | 否 | 不用缓存 | 二次编译加速；**与 debug 互斥** |

## 仅编译 Gameexe.dat

| 字段 | 填写 |
|---|---|
| 操作类型 | 仅编译 Gameexe.dat |
| 输入目录 | 含 Gameexe.ini 的目录 |
| 输出 | **必须**选文件夹 |

---

## 你会得到什么

- 指定的 `.pck` 文件（汉化场景包）
- 日志中的编译统计：场景数、宏、耗时等

---

## 常见错误

| 情况 | 处理 |
|---|---|
| 编码错误 | 源文件编码与 .ss 实际编码一致 |
| debug 与 tmp 同开 | 只保留一个 |
| 编译失败无明确行号 | 用 **分析** 对比原版；检查 .ss 是否改坏 |
| 极少数游戏 | 底部常量配置试 1 或 2 |
