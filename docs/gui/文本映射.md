# 文本映射 · 使用教程

汉化**核心工作流之一**：在脚本与 CSV 表格之间导出/写回译文，适合 Excel 协作翻译。

---

## 这个功能做什么

| 操作 | 作用 |
|---|---|
| 从 .ss 导出 | 把脚本中的可译文本导出为 `.ss.csv` |
| 写回 .ss | 把译好的 CSV 写回 `.ss` |
| 从 .dat 导出 | 从二进制 .dat 字符串池导出 |
| 写回 .dat | 写回已翻译的 .dat |

汉化推荐：**从 .ss 导出 → 翻译 → 写回 .ss → 编译**。

---

## 支持的主要文件

| 类型 | 说明 |
|---|---|
| `.ss` | SceneScript 源（decompiled 目录内） |
| 目录 | 含多个 .ss 或 .dat 的文件夹 |
| `.dat` | 编译后场景（dat 模式） |
| `.ss.csv` | 导出产生，写回时需与对应 .ss 配套 |

---

## 逐步操作（推荐 CSV 汉化）

### 1. 导出

1. 左侧 **文本映射**
2. **操作** → **从 .ss 导出**
3. **输入** → 选文件夹 → `汉化工作目录\output_xxx\decompiled`
4. **开始执行**
5. 在 decompiled 各目录下出现 `.ss.csv`

### 2. 翻译

1. 用 Excel / WPS 打开 CSV
2. **只修改译文/目标语言列**（勿改 ID、结构列）
3. **另存为 UTF-8 CSV**

### 3. 写回

1. **操作** → **写回 .ss**
2. **输入** → 同一 decompiled 目录
3. **开始执行**
4. .ss 文件中的文本被替换为 CSV 译文

### 4. 下一步

→ **编译** 生成 SceneZH.pck

---

## 字段说明

| 字段 | 支持什么 | 必填 | 结果 |
|---|---|---|---|
| 操作 | 四种互斥模式 | 必选 | 决定导出/写回目标 |
| 输入 | .ss / .dat / 目录 | **必填** | 处理对象 |

无单独输出路径：CSV 生成在输入旁；写回直接修改原 .ss。

---

## 你会得到什么

- **导出**：每个（或每组）.ss 对应 `.ss.csv`，含原文与待填译文列
- **写回**：原 .ss 被更新，可 diff 查看变化

---

## 常见错误

| 情况 | 处理 |
|---|---|
| CSV 乱码 | 保存为 UTF-8 |
| 写回无效 | 确认选「写回 .ss」且 CSV 与 .ss 文件名对应 |
| 破坏结构 | 不要删改 CSV 中的 ID/命令列 |
| dat 模式 | 高级流程；一般汉化用 .ss 即可 |

---

## 与「直接改 .ss」对比

| 方式 | 适合 |
|---|---|
| 文本映射 | 团队翻译、大量台词、非技术人员填表 |
| 直接改 .ss | 改动少、熟悉脚本格式 |

两种完成后都要 **编译**。
