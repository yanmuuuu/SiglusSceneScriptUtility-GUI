# 完整汉化工作流（做到底）

本文档描述 **从零到可运行汉化包** 的完整流程。假设游戏使用 **SiglusEngine**，且工具版本与游戏引擎在支持范围内。

---

## 第 0 步：准备

### 浏览游戏目录（可选）

**左侧：游戏资源 → 资源浏览**

1. 根目录选 **你的游戏目录**
2. 分类选 **脚本** 找到 `Scene.pck`，或 **图片** / **音频** 预览资源
3. 对 `.pck` 点 **跳转功能** 可直达「提取」

### 你需要准备

| 项目 | 说明 |
|---|---|
| 游戏本体 | 含 SiglusEngine.exe、Scene.pck（或同名场景包）、Gameexe.dat 等 |
| 空工作目录 | 例如「汉化工作目录」，用于存放解包结果 |
| 文本编辑器 | 改 .ss 或编辑 CSV（VS Code / Cursor 可配合「语言服务器」） |
| 备份 | **复制一份原版游戏文件夹**，补丁与编译出错时可恢复 |

### 首次打开本程序

1. 若弹出「缺少运行时常量」→ 点 **是**，进入 **初始化** → **开始执行**（需联网）
2. 日志显示成功后，再进行后续步骤

---

## 第 1 步：提取 Scene.pck

**左侧：场景与脚本 → 提取**

| 字段 | 填写 |
|---|---|
| 操作类型 | **提取 .pck** |
| 输入 | 选文件 → 你的游戏目录\Scene.pck |
| 输出目录 | 选文件夹 → 汉化工作目录（建议新建空文件夹） |
| 同时反汇编并反编译 | **建议勾选**（得到可编辑的 .ss 脚本） |
| 密钥来源 | 先留空；若提取失败再填 **你的游戏目录** |

点击 **开始执行**。

### 得到什么

在输出目录下出现类似 `output_日期_时间` 的文件夹，内含：

- 各场景 `.dat`、反编译的 `decompiled\*.ss`
- `Gameexe.ini`（若存在）
- 其他随包资源

**汉化主要改 `decompiled` 里的 .ss，或后续用文本映射生成的 CSV。**

### 提取失败时

- **密钥来源** 填你的游戏目录（需含 暗号.dat 或引擎文件）
- 见 **故障排除** 章节

---

## 第 2 步：翻译（二选一）

### 方式 A：直接改 .ss（适合熟悉脚本的人）

1. 用编辑器打开 `汉化工作目录\output_xxx\decompiled\` 下的 `.ss`
2. 修改对话文本，**保持引号、标签、命令格式不变**
3. 保存时编码与编译时「源文件编码」一致（汉化常用 **UTF-8**）

### 方式 B：文本映射 + CSV（推荐，适合表格翻译）

**左侧：场景与脚本 → 文本映射**

#### B1 导出

| 字段 | 填写 |
|---|---|
| 操作 | **从 .ss 导出** |
| 输入 | 选文件夹 → 上一步的 decompiled 目录（或单个 .ss） |

执行后，同目录生成 `.ss.csv` 文件。

#### B2 在 Excel / WPS 中翻译

- 打开 CSV，找到译文列（或按表头说明填写中文）
- **不要改 ID、行号、结构列**（只改允许翻译的文本列）
- 保存为 **UTF-8 CSV**

#### B3 写回

| 字段 | 填写 |
|---|---|
| 操作 | **写回 .ss** |
| 输入 | 同上 decompiled 目录（或对应 .ss） |

执行后，工具把 CSV 译文写回 `.ss` 文件。

---

## 第 3 步：编译回 .pck

**左侧：场景与脚本 → 编译**

| 字段 | 填写 |
|---|---|
| 操作类型 | **标准编译（.ss → .pck）** |
| 输入目录 | 汉化工作目录\output_xxx（含改好的 .ss 与 Gameexe.ini） |
| 输出 | 另存为 → 汉化工作目录\SceneZH.pck（文件名可自定） |
| 源文件编码 | **UTF-8**（若你用 UTF-8 保存脚本） |

点击 **开始执行**。日志末尾出现 `=== Compiling Stats ===` 表示编译完成。

### 得到什么

- 新的 `SceneZH.pck`（或你指定的名称）
- 这是汉化后的场景包，需让游戏加载它

### 编译失败时

- 检查 .ss 语法是否被破坏（多删了引号、标签）
- 尝试 **源文件编码** 改为 Shift-JIS（仅当你用该编码保存时）
- 底部 **常量配置** 改为 1 或 2 后重试（极少数引擎变体）
- 勾选 **纯 Python 模式** 排除 Rust 扩展问题（较慢）

---

## 第 4 步：让游戏使用汉化包

常见做法（因游戏而异）：

1. 将 `SceneZH.pck` 复制到你的游戏目录
2. 若游戏硬编码加载 `Scene.pck`：
   - 备份原版 `Scene.pck`
   - 将汉化包**改名**为 `Scene.pck` 替换，或
   - 使用下一步 **引擎补丁** 支持中文路径加载 `SceneZH.pck`

---

## 第 5 步：引擎补丁（中文显示 + 中文路径）

**左侧：系统与工具 → 引擎补丁**

**务必先备份 SiglusEngine.exe！**

### 5a 仅要界面中文

| 字段 | 填写 |
|---|---|
| 操作 | **CJK 语言 --lang cjk** |
| SiglusEngine.exe | 你的游戏目录\SiglusEngine.exe |
| 输出 exe | 另存为 → 你的游戏目录\SiglusEngine_zh.exe |

### 5b 要使用 SceneZH.pck 等中文文件名路径

| 字段 | 填写 |
|---|---|
| 操作 | **CJK + 中文路径 --lang cjk-path** |
| 输出 | 同上，生成新 exe |

### 5c 直接覆盖（慎用）

勾选 **直接覆盖原文件**，仅在已备份时使用。

### 得到什么

- 打了补丁的 exe，启动后界面为 CJK，并可识别中文路径的 pck（cjk-path 模式）

---

## 第 6 步：验证（强烈建议）

### 回编测试

**左侧：系统与工具 → 回编测试**

| 字段 | 填写 |
|---|---|
| 输入 | 你的游戏目录\Scene.pck（原版，用于测工具链） |

自动执行 提取→编译→对比。日志显示通过则说明环境正常。

### 游戏内验证

1. 用补丁后的 exe 启动
2. 检查菜单、对话是否中文
3. 抽查几个场景分支、选项、人名显示

### 对比原版与汉化版（可选）

**左侧：场景与脚本 → 分析 → 比较两个文件**

- 输入 1：原版 Scene.pck
- 输入 2：SceneZH.pck
- 可选 **语义级比较** 查看文本级差异

## 汉化检查清单

- [ ] 已初始化 const.py
- [ ] 已备份原版游戏
- [ ] 提取成功，有 decompiled\.ss
- [ ] 译文已写回或 .ss 已手改
- [ ] 编译成功，有 SceneZH.pck
- [ ] 引擎补丁已打（如需中文 UI / 中文路径）
- [ ] 进游戏实测主线 + 若干分支
- [ ] 无乱码、无闪退、无未翻译残留（按项目要求）

---

## 最短路径回顾

```
初始化 → 提取(勾选反编译) → 文本映射导出 → 翻译CSV → 写回 → 编译 → 引擎补丁 → 进游戏测试
```

任一步骤的字段说明，请点该功能面板右上角 **「教程」**，或在本手册左侧选择对应章节。
