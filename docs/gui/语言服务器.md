# 语言服务器 · 使用教程

> **便携版用户：** GUI 可单独启动；LSP 仍用便携包内 CLI，见 [instructions.md](../../instructions.md#便携版推荐)。

启动 **SiglusSS LSP**，在 VS Code / Cursor 中编辑 `.ss` 时获得补全、诊断、跳转（汉化改脚本时可选）。

---

## 这个功能做什么

- 后台运行 `siglus-ssu -lsp`
- 编辑器通过 stdio 连接后提供语法服务
- **不**直接修改游戏文件

---

## 何时使用

| 适合 | 不适合 |
|---|---|
| 大量手改 .ss | 只用 CSV 文本映射 |
| 需要跳转定义、报错提示 | 只点 GUI 提取编译 |

---

## 逐步操作

1. **系统与工具** → **语言服务器**
2. 可选 **禁用并行扫描（--serial）**（慢机或大工程）
3. 点 **启动 LSP**（非「开始执行」文案，同一按钮）
4. 日志显示已启动后，**保持本程序运行**
5. 在编辑器配置 LSP 命令指向 `siglus-ssu -lsp`（stdio）
6. 结束时点 **停止**

---

## 字段说明

| 字段 | 说明 |
|---|---|
| 禁用并行扫描 --serial | 大工作区扫描时更稳但更慢 |

无路径字段。

---

## 编辑器配置概要

- 命令：`siglus-ssu -lsp`（便携版用自带 CLI 路径）
- 传输：stdio
- 工作区：打开你的 `汉化工作目录\output_xxx\decompiled` 文件夹

详细见 **manual_cn.md** LSP 章节。

---

## 汉化工作流中的位置

可选步骤，与 **直接改 .ss** 并行：

```
提取 → [开 LSP 改 .ss] 或 [文本映射] → 编译
```

---

## 常见错误

| 情况 | 处理 |
|---|---|
| 编辑器无提示 | 确认 GUI 中 LSP 仍在运行；路径配置正确 |
| 扫描卡死 | 勾选 serial；缩小打开文件夹范围 |
| 重复启动 | 日志提示已在运行时点停止后再开 |
